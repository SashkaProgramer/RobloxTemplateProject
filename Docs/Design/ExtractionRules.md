# Extraction Rules

## Design Purpose

Extraction is the emotional climax of the match. It should convert temporary cooperation into pressure, suspicion, and final conflict.

## Core Rules

- Extraction becomes available only during the final phase.
- Not all players can extract.
- Only extracted players secure carried run loot.
- A player who dies before extraction loses carried run loot.

## MVP Rule Set

- One extraction point for the full match
- Limited extraction slots
- Limited extraction window
- Server-authoritative resolution of who extracted successfully

## Tension Sources

- Players do not all fit in the final evacuation.
- Valuable loot increases the stakes of who should survive.
- The route to extraction is still dangerous.
- Sabotage remains possible close to the end.

## Key Design Questions

- How many slots should exist relative to player count?
- Should extraction require a shared activation step?
- How much warning should players get before extraction closes?
- How visible should remaining extraction capacity be?

## MVP Guidance

- Keep extraction rules simple and brutal.
- Avoid hidden rule complexity.
- Make the outcome readable so losses feel harsh but understandable.
